NIST AI Risk Management Framework crosswalk

How aimissionready.org guides map to the four NIST AI RMF functions

The NIST AI Risk Management Framework (AI RMF) is the federal reference standard for AI governance. It is increasingly cited by federal funders, state regulators, and sophisticated foundation grantmakers. The framework has four core functions: Govern, Map, Measure, and Manage. This crosswalk shows where the aimissionready guides cover each function, so that nonprofits using these resources can demonstrate framework alignment when funders or auditors ask. It is not a certification; it is a navigation tool.

NIST AI RMF · Function 1

GOVERN — Cultivate a culture of risk management

NIST's GOVERN function covers the organizational policies, accountability structures, and culture that make all the other functions possible. For nonprofits, this maps closely to the policy, governance, and board-level work that creates the conditions for everything else.

NIST sub-categories particularly relevant to nonprofits: GOVERN 1.1 (policies and processes), GOVERN 2.1 (roles and responsibilities), GOVERN 3.2 (board awareness and oversight), GOVERN 4.1 (organizational culture), GOVERN 6.1 (third-party risk management).

NIST AI RMF · Function 2

MAP — Establish context and identify risks

NIST's MAP function covers understanding the AI system's purpose, the context of its use, the affected populations, and the risks specific to each. For nonprofits, this is the equity-centered work that asks who benefits, who is at risk, and what the actual use case is.

NIST sub-categories particularly relevant to nonprofits: MAP 1.1 (system context), MAP 2.2 (purpose and use), MAP 3.1 (benefits and costs), MAP 4.1 (impact assessment), MAP 5.1 (affected groups).

NIST AI RMF · Function 3

MEASURE — Analyze and assess identified risks

NIST's MEASURE function covers the assessment, testing, and monitoring activities that turn identified risks into managed risks. For most nonprofits, this is lighter-weight than NIST's enterprise framing assumes, but the principle holds: you cannot manage what you do not measure.

NIST sub-categories particularly relevant to nonprofits: MEASURE 1.1 (test and evaluation), MEASURE 2.6 (bias and fairness), MEASURE 2.10 (privacy), MEASURE 3.1 (effectiveness), MEASURE 4.1 (continuous monitoring).

NIST AI RMF · Function 4

MANAGE — Allocate resources to mapped and measured risks

NIST's MANAGE function covers the operational response: prioritizing identified risks, implementing controls, responding to incidents, and reviewing decisions. For nonprofits, this is the day-to-day work of running an AI-aware organization.

NIST sub-categories particularly relevant to nonprofits: MANAGE 1.1 (risk prioritization), MANAGE 2.2 (third-party management), MANAGE 3.1 (incident response), MANAGE 4.1 (continuous improvement).

How to use this crosswalk

If a funder asks whether your AI governance work aligns with NIST AI RMF, this crosswalk gives you a defensible answer: "We use the aimissionready framework, which maps to all four NIST AI RMF functions as follows..." and you can hand them this page. It is not an audit or a certification, but it is a credible navigation showing that the work you have done covers the same territory NIST covers.

For organizations subject to actual NIST AI RMF compliance requirements (federal grant programs, certain state regulators, some federal contractors), this crosswalk is a starting point, not a substitute for the full NIST framework documentation. Read NIST AI RMF 1.0 directly when the stakes warrant it. Linked at the External Resources page under Frameworks and Standards.

This crosswalk was last reviewed against NIST AI RMF 1.0. NIST has signaled updates to come. When NIST AI RMF 2.0 lands, this document will be revised to match.
